If…Then & Biconditionals

Name:   Date:

Conditionals — “if…then”

A conditional has the form if p, then q, written p → q.

  • p = hypothesis (antecedent) — the “if” part.
  • q = conclusion (consequent) — the “then” part.
Promise metaphor

“If you finish your chores, then you get dessert.” Think of p → q as a promise. The promise is broken only when chores are done (p true) but dessert is denied (q false).

If chores are not done (p false), the promise is not broken — dessert might still happen for another reason, and the “if…then” can still count as true in logic.

Only one false row: p → q is false exactly when a true hypothesis leads to a false conclusion. Memorize that row: T → F = F.

Converse, inverse, contrapositive

Start with: “If a number is divisible by 4, then it is even.” Let p = divisible by 4, q = even.

Form Symbol Sentence True?
Originalp → qIf divisible by 4, then even.Yes
Converseq → pIf even, then divisible by 4.No (6)
Inverse¬p → ¬qIf not div. by 4, then not even.No (6)
Contrapositive¬q → ¬pIf not even, then not div. by 4.Yes
Big takeaway A conditional and its contrapositive always have the same truth value. The converse and inverse need their own checks — do not assume they match the original.

Biconditional — “if and only if”

p ↔ q means both p → q and q → p. Both directions must hold.

Example

“A triangle is equilateral if and only if it has three equal angles.” Being equilateral and having three equal angles are equivalent — each implies the other.
